Wake up kata   ,
Organisation,
Formez des binômes.

Dans chaque binôme :

une personne écrit la fonction demandée,,
l’autre personne écrit les tests associés.,

Étapes à suivre,
Prenez connaissance de l’énoncé de l’algorithme.,
Répartissez les rôles :
une personne développe la fonction,,
une personne rédige les tests.,
,
Travaillez séparément sur vos fichiers.,
Une fois terminé, échangez vos codes.,
Lancez les tests.,
Vérifiez si les tests passent.,

Analyse des résultats,
Si les tests passent, comparez votre raisonnement et vérifiez que la fonction couvre bien les cas attendus.

Si les tests ne passent pas, prenez le temps d’identifier l’origine du problème :

la fonction ne respecte-t-elle pas exactement l’énoncé ?,
les tests vérifient-ils les bons comportements ?,
certains cas particuliers ont-ils été oubliés ?,
le nom de la fonction, les paramètres ou la valeur retournée correspondent-ils bien à ce qui était attendu ?,

Consigne de l'algo,
Créer une fonction isPalindrome(text) qui retourne :

true si le texte est un palindrome ;,
false sinon.,

Un palindrome est une chaîne de caractères qui se lit de la même façon de gauche à droite et de droite à gauche.

---

Contraintes,
si le paramètre text est null ou undefined, ou ne contient pas de caractère, retourner false,
si le paramètre text n’est pas une chaîne de caractères, retourner false,
ignorer la casse (majuscules / minuscules),
on considère qu’il n’y a ni espaces ni caractères spéciaux dans la chaîne,
la fonction doit retourner un booléen,

---

Exemples attendus,
isPalindrome("kayak") → true,
isPalindrome("radar") → true,
isPalindrome("Kayak") → true,
isPalindrome("bonjour") → false,
isPalindrome("") → false,
isPalindrome(null) → false,
isPalindrome(123) → false
